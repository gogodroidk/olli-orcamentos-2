# QA artifacts

`qa-artifacts/` is an ignored local folder used by QA scripts and manual visual checks.

It is intentionally not part of the tracked product source. Keep files there while they are useful for visual verification, and move only release-grade historical evidence into `docs/archive`.

Current local categories:

- `qa-artifacts/web/`: generated by `npm run qa:web` and related web/mobile checks.
- `qa-artifacts/iphone-lab/`: generated during iPhone Lab checks.
- `qa-artifacts/handoff/`: handoff visual evidence.
- `qa-artifacts/android/`: Android viewport checks.
- `*.log`: local server or preview logs.

The authoritative script output path is `scripts/qa-web.mjs`, which writes screenshots to `qa-artifacts/`.

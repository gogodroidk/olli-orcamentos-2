# Documentation index

Start here when you need to understand the project without scanning every folder.

## Product and code

- `PROJECT_STRUCTURE.md`: layer-by-layer structure and conventions.
- `ORGANIZATION.md`: where files belong and what should stay archived or ignored.
- `CODE_MAP.md`: practical map of app, worker, scripts, database and larger files.
- `FILE_INDEX.md`: generated file-by-file index for tracked/new project files.
- `LOCAL_OUTPUTS.md`: ignored local outputs present on this machine.
- `SUPABASE.md`: backend, RLS and Supabase notes.

## Operations

- `../README.md`: setup and daily commands.
- `../LEIA-ANDROID.md`: Android / Play Store publishing guide.
- `../worker/README.md`: Cloudflare Worker guide.
- `QA_ARTIFACTS.md`: how to read local visual QA outputs.

## Plans and archive

- `superpowers/plans/`: execution plans and handoff notes.
- `archive/`: historical screenshots, tool logs, snapshots and other evidence moved out of active folders.

## Current source of truth

- Runtime app code is in `../src`.
- Worker code is in `../worker`.
- Runtime image/data assets are in `../assets`.
- Generated local outputs are ignored unless deliberately archived with a README or manifest.

# Project organization

This repository is organized around the app first, with generated evidence and tool output moved out of the root.

## Root files

Keep only files that are needed to understand, install, build, or publish the app:

- `README.md`: project overview and first commands.
- `AGENTS.md`: mandatory Codex/agent instruction for Expo SDK 56.
- `package.json` and `package-lock.json`: Node/npm dependencies and scripts.
- `app.json`, `eas.json`, `metro.config.js`, `tsconfig.json`, `index.ts`, `App.tsx`: Expo and app entry/config files.
- `.env.example`: public environment template.
- `.mcp.json`, `.claude/settings.json`, `.codex/config.toml`: local agent/tool configuration.

## Product code

- `src/components`: reusable UI.
- `src/screens`: full app screens.
- `src/steps`: quote wizard steps.
- `src/navigation`: app routes and navigation helpers.
- `src/services`: Supabase, links, sync, AI and integration code.
- `src/database`: SQLite persistence, backups, restore and metrics.
- `src/theme`: visual tokens and fonts.
- `src/types`: domain contracts.
- `src/utils`: formatting, PDF, image and pure helpers.
- `worker`: Cloudflare Worker for public links, AI and admin surface.
- `supabase/migrations`: database schema and RLS history.

## Assets

- `assets`: runtime assets used by the app. Do not move these without updating `app.json` and imports.
- `public`: Cloudflare/static web deployment headers and redirects.
- `preview/iphone-lab.html`: source template used by `scripts/iphone-lab.mjs`; it is not an archive file.

## QA and generated output

- `qa-artifacts`: ignored local QA screenshots and logs generated by `npm run qa:web` and iPhone Lab.
- `dist`: ignored web export output.
- `android` and `ios`: ignored generated native folders.
- `.expo`: ignored Expo cache and local state.
- `.playwright-mcp`: ignored future Playwright MCP captures. Historical tracked captures were moved to `docs/archive/tool-runs/playwright-mcp/`.
- `.codex/*.log`: ignored future Codex run logs. Historical logs were moved to `docs/archive/tool-runs/codex/`.
- `supabase/.temp`: ignored future Supabase CLI local state. The previously tracked CLI marker was moved to `docs/archive/supabase/`.

## Archive policy

Archive means "kept, documented, and out of the way"; it does not mean disposable. Nothing was intentionally deleted during the organization pass.

Use `docs/archive` for historical evidence that should stay available but should not sit in the root or active tool folders.

## Safety rules

- Do not move files from `assets`, `src`, `worker`, `scripts`, `preview`, `public`, `supabase/migrations`, or root config files unless imports/config references are updated in the same change.
- Generated screenshots and logs should be described by a README or manifest before they are kept long term.
- Secrets stay out of tracked files. `.env.local` remains ignored.
